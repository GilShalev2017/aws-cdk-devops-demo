CDK Stacks Cheat Sheet - Deployment Actions

1. StorageStack - Foundation Layer

Purpose: Creates data storage resources

// Main Actions:
- Creates DynamoDB tables (products, orders, users) with on-demand billing
- Creates S3 bucket for file uploads (assets)
- Sets up removal policies for demo cleanup
- Outputs table names and bucket ARN for other stacks

Key Resources:

  • DynamoDBTable ×3 (products, orders, users)
  • S3 Bucket (assets storage)

2. LambdaStack - Business Logic Layer

Purpose: Deploys microservice functions

// Main Actions:
- Creates Lambda functions (products, orders, users services)
- Grants IAM permissions: read/write to DynamoDB tables
- Grants S3 permissions (products Lambda needs file upload access)
- Uses common configuration (runtime, memory, timeout)
- Outputs function ARNs for API Gateway integration

Key Resources:

  • Lambda Function ×3 (Node.js 18.x, 256MB, 30s timeout)
  • IAM Policies (DynamoDB + S3 access)

3. ApiStack - API Layer

Purpose: Creates REST API endpoints

// Main Actions:
- Creates API Gateway REST API with stage 'v1'
- Sets up CloudWatch logging role
- Configures CORS for all origins
- Creates resource structure: /api/{products,orders,users}/{id}
- Integrates each endpoint with corresponding Lambda
- Outputs API URL and endpoint URLs

Key Resources:

  • REST API (stage: v1)
  • Lambda Integrations ×3
  • CORS Configuration
  • CloudWatch Role

API Routes:

GET/POST    /api/products
GET/PUT/DEL /api/products/{id}
GET          /api/products/{id}/upload-url

GET/POST    /api/orders  
GET/PUT/DEL /api/orders/{id}

GET/POST    /api/users
GET/PUT/DEL /api/users/{id}

4. FrontendStack - Presentation Layer

Purpose: Deploys React app with CDN

// Main Actions:
- Creates private S3 bucket for static hosting
- Creates Origin Access Control (OAC) for secure S3 access
- Creates CloudFront distribution with:
  * S3 origin for static files (React app)
  * HTTP origin for API Gateway proxy (/api/*)
  * Custom error handling (SPA routing)
- Adds bucket policy for CloudFront OAC access
- Deploys React build files to S3
- Outputs CloudFront URL

Key Resources:

  • S3 Bucket (static hosting)
  • CloudFront Distribution (CDN + API proxy)
  • Origin Access Control (secure S3 access)
  • Bucket Deployment (React build sync)

CloudFront Behaviors:

Default: S3 → React static files
/api/*:   HTTP Origin → API Gateway (stage v1)

Deployment Order (Dependencies)

1. StorageStack (no dependencies)
2. LambdaStack (depends on StorageStack)
3. ApiStack (depends on LambdaStack)  
4. FrontendStack (depends on ApiStack)

Key Integration Points

  • Storage → Lambda: Table names passed as props
  • Lambda → API: Function ARNs used for integrations
  • API → Frontend: API URL passed for CloudFront proxy
  • Frontend → S3: React build deployed via BucketDeployment

Architecture Pattern

3-Tier Serverless Architecture:

  • Data Layer: DynamoDB + S3
  • Logic Layer: Lambda microservices
  • Presentation Layer: React + CloudFront + API Gateway

Perfect for DevOps interview - shows understanding of serverless patterns, CI/CD, infrastructure as code!
